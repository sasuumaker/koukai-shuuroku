# 非エンジニア向け個人開発基礎講座 - 設計ドキュメント

## 概要

非エンジニアが「コードを見ずに」プロダクトを開発・デプロイ・運用できる状態を目指す基礎講座。
.AIとしての非エンジニア向け開発ベストプラクティスの土台となる。

## 作るもの

**TODOアプリ**（CRUD + 認証の基本が網羅できる）

## 技術スタック

| 技術 | 用途 |
|------|------|
| Next.js（App Router） | フロントエンド |
| Supabase（MCP連携） | バックエンド・DB・認証 |
| Tailwind CSS | スタイリング |
| shadcn/ui | UIコンポーネント |
| Claude Code（ローカル） | 開発エージェント |
| Claude Code Web | 修正・PR・マージ |
| GitHub | ソースコード管理 |
| Vercel | ホスティング・デプロイ |

## コアの思想

- **コードは見なくてOK**。見るのはenvファイル更新くらい
- 全会員がこの前提を理解できている状態にする
- ローカルClaude Code = 管理者権限（構築・デプロイ）
- Claude Code Web = PR経由の安全な修正（委任・スマホ対応）

## Claude Codeのプラン要件

| プラン | 可否 | 備考 |
|--------|------|------|
| MAX（推奨） | ○ | 講座はMAX前提で進行 |
| Pro | △ | トークン上限に引っかかる可能性あり |
| 代替サービス | △ | MYUUU等のプロキシサービスも選択肢として紹介 |

## 無料枠の調査結果

### Supabase Free Tier

| 項目 | 無料枠 | TODOアプリで足りるか |
|------|--------|---------------------|
| プロジェクト数 | 2つ | ○ 1つで十分 |
| DB容量 | 500 MB | ○ TODOデータなら余裕 |
| APIリクエスト | 無制限 | ○ |
| 認証ユーザー数 | 50,000 MAU | ○ 講座レベルなら余裕 |
| ファイルストレージ | 1 GB | ○ |
| Edge Functions | 500,000回/月 | ○ |
| 帯域幅 | 10 GB/月 | ○ |

**注意点**: 7日間アクセスなしでプロジェクトが自動停止（pause）される。講座後に放置すると止まる。再開はダッシュボードからワンクリック。

**結論: TODOアプリなら無料枠で全く問題なし。**

### Vercel Hobby（無料）プラン

| 項目 | 無料枠 | TODOアプリで足りるか |
|------|--------|---------------------|
| 帯域幅 | 100 GB/月 | ○ 個人TODOなら余裕 |
| Serverless Functions | 12個/デプロイ | ○ TODOアプリなら十分 |
| 実行タイムアウト | 60秒 | ○ |
| プレビューデプロイ | ○ | ○ PR連携で自動プレビュー |
| チーム利用 | × | 個人利用のみ |

**注意点**: 商用利用は不可（Hobbyプランの規約）。あくまで学習・個人プロジェクト用。本格運用するならPro（月$20）へ。

**結論: 講座レベルなら無料枠で問題なし。**

## フェーズ構成

### フェーズ1: プロトタイプ開発 → Vercelデプロイ

**使用ツール**: ローカルClaude Code（Cursorのターミナル or その他ターミナル）

**受講者がやること**:
- [ ] GitHubアカウント作成・リポジトリ作成
- [ ] Vercelアカウント作成・GitHub連携
- [ ] Supabaseプロジェクト作成
- [ ] Claude Codeインストール・初期設定
- [ ] envファイル設定（Supabase URL, API Key等）
- [ ] Claude Codeに指示してTODOアプリ生成
- [ ] GitHubにpush → Vercel自動デプロイ
- [ ] 動作確認

**受講者が見るコード**: envファイルのみ

### フェーズ2: デプロイ後の修正

**使用ツール**: Claude Code Web

**受講者がやること**:
- [ ] Claude Code WebでGitHubリポジトリ接続
- [ ] 修正指示を自然言語で入力
- [ ] PR作成 → Vercelプレビューで確認
- [ ] マージ → 本番自動デプロイ

**セキュリティの教え方**:
> ローカルは自分の家の鍵を全部持ってる状態。
> Webは玄関のインターホンまでしか入れない状態。
> 提案（PR）はできるけど、家の中をいじるかどうかは自分が決める。

## 所要時間

基礎講座はその他講座（最大2日）より長くてもよい。
全ステップを講座内でカバーする（事前課題として切り出さない）。

| 項目 | 想定時間 | 備考 |
|------|----------|------|
| 環境構築（アカウント作成等） | 未定 | 講座内でやる |
| フェーズ1（開発→デプロイ） | 未定 | |
| フェーズ2（Web修正→マージ） | 未定 | |
| 合計 | 未定 | 実際に作ってみて計測する |
